Namespaces

In SilverGarden, Your program containing a billion source files will
turn into a single SilverLib file. which then can be turned into a runnable program.

Avoiding a mess with namespaces

Now you can probably imagine the mess it would be.
but silver garden comes with a fancy way of avoid that mess. which is
namespaces!

its only your ordinary namespaces from C#, TS and others, but,
unlike these languages, you need to specify your namespace with a string literal. like this:

		namespace "My.New.Fancy.Namespace"
Good Pratice time!

Avoid using the Default Namespace

The default namespace is a unsafe place, that is
only there to be used for objects that are located in the
target export language! so, don't use it as your namespace >:(

Imports

Of course, namespaces would be unusable without imports.
an import statment in SilverGarden requires a string parameter,
(its exacly the same syntax as the namespace statment)

Import Example

		import "ImportMe.Namespace"

Extern Imports

SilverGarden provides a basic "multilanguage" standard library,
which is simply not enough, so, you have extern imports.
Extern Imports change from target language to target language, for example,
if the target language is C++, it will be turned into a include preprocessor instruction.

The Extern Import Syntax

its exacly like the normal import syntax, but the difference is that
a "extern" keyword is required before the object.

		import extern "stdio.h"
Good Pratice time!

Use LanguageChecks!

if you want to avoid compilation errors in the backend of your
silver Garden library, use language checks with your extern imports.
